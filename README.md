# 🇮🇶 Iraq Babylon Discord Bot

بوت إدارة وحماية احترافي لسيرفر Discord، بواجهة عربية كاملة، مبني بـ **Discord.js v14**.

## ✨ الميزات المُنفّذة في هذا الكود

- **أوامر عامة:** `/مساعدة` `/بنق` `/معلومات_البوت` `/معلومات_السيرفر` `/الإحصائيات`
- **إدارة كاملة:** حظر، فك حظر، طرد، كتم (رتبة)، فك كتم، تايم أوت، إزالة تايم أوت، تحذير، عرض/إزالة التحذيرات، حذف رسائل، قفل/فتح روم، Slowmode، إضافة/إزالة رتبة، نقل/سحب/فصل عضو صوتياً، إعلان، برودكاست، إعادة تحميل، إعادة تشغيل، Backup/Restore
- **نظام تذاكر متكامل:** لوحة ثابتة تُعاد تلقائياً إذا انحذفت، اختيار نوع التذكرة، Modal لكتابة السبب، Claim/Unclaim/Close/Reopen/Delete/Rename/إضافة وإزالة أعضاء، Transcript بصيغة HTML
- **أنظمة حماية:** Anti Raid (كشف الانضمامات الجماعية)، Anti Spam، Anti Links، Anti Discord Invite، Anti Everyone/Here، Anti Mention Spam، Anti Webhook، Anti Bot Add، Anti Mass Join، Anti Nuke لحذف/تعديل الرومات والرتب، Anti Permission Abuse، Anti Nickname Spam، حماية الحسابات الجديدة — كل نظام يُفعَّل/يُعطَّل من لوحة الإعدادات
- **لوقات شاملة:** Moderation / Chat (حذف وتعديل) / Voice / Security / Ticket / Join-Leave
- **لوحة إعدادات تفاعلية بالكامل بالأزرار والقوائم المنسدلة:** `/الاعدادات`
- **صلاحيات:** كل أوامر الإدارة مقيدة برتبة `OWNER_ROLE_ID` (أو مالك السيرفر) فقط
- **قاعدة بيانات JSON** بسيطة بدون الحاجة لتثبيت أي سيرفر خارجي (Users/Tickets/Warns/Config/Logs/Backups)

## 📦 التثبيت

```bash
npm install
cp .env.example .env
# عدّل .env وضع فيه التوكن، CLIENT_ID، و OWNER_ROLE_ID على الأقل
npm run deploy   # ينشر أوامر السلاش على ديسكورد
npm start        # يشغّل البوت
```

### إعداد `.env`

| المتغير | الوصف |
|---|---|
| `TOKEN` | توكن البوت من [Discord Developer Portal](https://discord.com/developers/applications) |
| `CLIENT_ID` | آيدي التطبيق |
| `GUILD_ID` | (اختياري) آيدي سيرفر تجريبي لنشر الأوامر فوراً بدلاً من الانتظار ساعة |
| `OWNER_ROLE_ID` | رتبة المالك التي تتحكم بكل أوامر الإدارة |
| `OWNER_USER_ID` | (اختياري) آيدي شخص يملك صلاحية مطلقة حتى بدون الرتبة |

### صلاحيات البوت المطلوبة عند الدعوة (Invite)

عند إنشاء رابط الدعوة من الـ Developer Portal، فعّل الصلاحيات التالية على الأقل:
`Manage Roles, Manage Channels, Kick Members, Ban Members, Moderate Members, Manage Messages, Manage Webhooks, Manage Nicknames, Read Message History, Send Messages, View Audit Log, Connect, Move Members`
بالإضافة إلى الـ Scopes: `bot` و `applications.commands`.

كما يجب تفعيل الـ **Privileged Gateway Intents** التالية من الـ Developer Portal: `Server Members Intent` و `Message Content Intent`.

## ⚙️ الإعداد الأولي داخل السيرفر

1. شغّل الأمر `/الاعدادات` (بعد أن تملك رتبة `OWNER_ROLE_ID`)
2. حدّد روم اللوقات، روم التذاكر، رتبة الإدارة، والرتبة التلقائية من الأزرار
3. من زر "أنظمة الحماية" فعّل/عطّل ما تريد
4. استخدم `/لوحة_تذاكر` لنشر لوحة فتح التذاكر في روم مخصص (بديل سريع عن الزر بالإعدادات)

## 📂 هيكل المشروع

```
iraq-babylon-bot/
├── index.js                  # نقطة التشغيل
├── config.js                 # الإعدادات الثابتة (ألوان، حدود الحماية، إلخ)
├── commands/
│   ├── general/               # /مساعدة /بنق /معلومات_البوت ...
│   ├── admin/                 # كل أوامر الإدارة والحماية اليدوية
│   ├── tickets/                # /تذكرة /لوحة_تذاكر
│   └── settings/               # /الاعدادات
├── events/                    # كل أحداث Discord.js (رسائل، أعضاء، رومات، رتب...)
├── handlers/                  # محمّلات الأوامر/الأحداث + منطق الأزرار/القوائم/المودالز/التذاكر
├── systems/                   # أنظمة الحماية (Anti-Spam, Anti-Nuke, Anti-Links)
├── database/                  # طبقة قاعدة بيانات JSON + إعدادات كل سيرفر
└── utils/                     # Embeds موحدة، صلاحيات، لوقر
```

## 🛠️ ملاحظات تقنية مهمة

- **قاعدة البيانات:** ملفات JSON داخل `database/data/` — مناسبة لسيرفر واحد أو عدد محدود من السيرفرات. لعدد كبير من السيرفرات يُفضّل الترقية إلى MongoDB أو SQLite لاحقاً (الكود مصمم بطبقة `db.js` بسيطة يسهل استبدالها).
- **الكتم:** أمر `/كتم` يستخدم رتبة `Muted` (يُنشئها البوت تلقائياً ويضبط صلاحياتها بكل الرومات)، بينما `/تايم_اوت` يستخدم ميزة Discord Timeout الأصلية.
- **Anti-Nuke:** يراقب حذف/تعديل الرومات والرتب عبر Audit Log، وإذا كرر نفس الشخص 3 أفعال خطيرة خلال 10 ثواني يتم حظره/طرده تلقائياً (باستثناء Owner).
- **Backup/Restore:** ينسخ أسماء الرومات والرتب وصلاحياتها الأساسية وإعدادات البوت. عند الاستعادة يُنشئ فقط ما هو ناقص (لا يحذف أو يستبدل الموجود) تفادياً لأي كارثة غير مقصودة.
- عدّل `config.js` لتغيير الألوان، حدود السبام، حد الـ Anti-Nuke، وعدد أيام حماية الحسابات الجديدة.

## 🚀 التشغيل الدائم (Production)

يُنصح باستخدام [PM2](https://pm2.keymetrics.io) لتشغيل البوت بشكل دائم وإعادة تشغيله تلقائياً:

```bash
npm install -g pm2
pm2 start index.js --name iraq-babylon-bot
pm2 save
```

## 🆕 تحديث: نظام الحماية Pro Max

هذه الميزات أُضيفت فوق الكود الأصلي بالكامل دون حذف أي أمر أو ملف موجود مسبقاً:

### 🚫 Anti-Spam مرحلي (تحذير ثم عقوبة)
- إذا أرسل عضو 5 رسائل متكررة خلال فترة قصيرة (`config.spam.maxMessages` / `intervalMs`) → **تحذير فقط** بدون حذف أو كتم.
- إذا استمر بالسبام بعد التحذير → **حذف رسائله المتكررة** بالروم تلقائياً + **تايم أوت لمدة ساعة كاملة** (`config.spam.escalateTimeoutMinutes`).
- إذا مرت `warnResetMs` (30 ثانية افتراضياً) دون تكرار السبام، تُعاد حالته للصفر ويُعامل كأول مرة بالمرة القادمة.

### 🛡️ Anti-Nuke Pro Max (تصفير الرتب بدل الباند/الكيك)
- عند تجاوز أي شخص **حتى لو كان أدمن أو صاحب رتبة عالية** للحد المسموح من الأفعال الخطيرة (حذف/إنشاء رومات، حذف/إنشاء رتب، منح صلاحية Administrator، إنشاء Webhook، حظر/طرد/تايم أوت أعضاء)، يقوم البوت تلقائياً بـ **تصفير جميع رتبه** بدلاً من حظره أو طرده من السيرفر.
- **المالك (Owner) فقط** معفى تلقائياً من هذا النظام.
- كل نوع فعل له **حد مستقل خاص به** (وليس عداد واحد مشترك)، ويمكن التحكم بكل حد على حدة من كل سيرفر عبر الأمر:

  ```
  /حدود_الحماية عرض        → عرض كل الحدود الحالية
  /حدود_الحماية تعيين      → تحديد حد أقصى لنوع فعل معين (مثلاً: كم رتبة يحذف قبل العقاب)
  /حدود_الحماية المدة       → تحديد الفترة الزمنية بالثواني التي تُحتسب خلالها الأفعال
  ```

- الأنواع القابلة للتحكم بها: حذف/إنشاء/تعديل رومات، حذف/إنشاء/تعديل رتب، إنشاء Webhook، منح صلاحية Administrator، حظر أعضاء (Ban)، طرد أعضاء (Kick)، تايم أوت لأعضاء.
- تمت إضافة مراقبة جديدة لعمليات **الحظر الجماعي** (`events/guildBanAdd.js`)، **الطرد** (تم تحديث `events/guildMemberRemove.js`)، و**التايم أوت المتكرر** (تم تحديث `events/guildMemberUpdate.js`) — بحيث أي أدمن يسيء استخدام صلاحياته يتحاسب تلقائياً بنفس آلية Anti-Nuke.

### 🔒 البوت أصبح حصرياً للأونر فقط
- تم إضافة فحص عام (`events/interactionCreate.js`) يمنع **أي شخص عدا الأونر** (رتبة/آيدي Owner المحدد بالإعدادات، أو مالك السيرفر) من استخدام **أي أمر سلاش بالبوت إطلاقاً** — حتى الأوامر العامة كـ `/بنق` و `/مساعدة`.
- نظام التذاكر (فتح تذكرة عبر الزر) بقي متاحاً لجميع الأعضاء كما هو، لأنه جزء من خدمة الدعم للأعضاء وليس أمر إدارة.

### 🔊 أوامر إضافية للروم الصوتي
- `/قفل_الصوت` — قفل روم صوتي بحيث لا يستطيع أحد الدخول إليه.
- `/فتح_الصوت` — إعادة فتح الروم الصوتي بعد قفله.
- (الأوامر الصوتية الموجودة مسبقاً بقيت كما هي: `/نقل_عضو`, `/سحب_عضو`, `/فصل_عضو`).
